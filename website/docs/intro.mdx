# Introdução

Nesse guia, aprenderemos a utilizar o JS SDK do Starlight, uma ferramenta oficial que automatiza o processo
de integração de sites e aplicações JavaScript com as APIs do Starlight.

:::tip Dica

Quer ver exemplos de uso ou lembrar rapidamente como utilizar uma API específica? Dê uma olhada na seção Cheatsheet.

:::

O JS SDK pode ser usado em qualquer aplicação JavaScript e TypeScript. Porém, se sua aplicação usa React, você pode
utilizar o Starlight React SDK, que provê componentes React que facilitam a integração com o Starlight, além de todas
as funcionalidades do JS SDK.

:::info Versão da API

O JS SDK requisita informações do Starlight utilizando a versão 2 da Query API.

:::

## Instalação

Para começar a utilizar o JS SDK, instale-o no seu projeto:

<Tabs>
  <TabItem value="npm" label="npm" default>

  ```bash
  npm install @starlightcms/js-sdk
  ```

  </TabItem>
  <TabItem value="yarn" label="yarn" default>

  ```bash
  yarn add @starlightcms/js-sdk
  ```

  </TabItem>
</Tabs>

## Configuração

Antes de começar a utilizar o JS SDK, você precisa configurar qual área de trabalho deve ser utilizada para requisitar
dados. Utilize o método `configure` para isso:

```js
import Starlight from '@starlightcms/js-sdk'

Starlight.configure({
  workspace: '1234567890'
})
```

A configuração do Starlight só precisa ser feita uma vez, idealmente no início do ciclo de vida da sua aplicação.

:::danger Identificador

Atualmente, a versão 2 das APIs do Starlight só identificam áreas de trabalho utilizando seus IDs. O suporte à slugs
ainda não foi implementado.

Você pode pegar o ID de uma área de trabalho na interface de administração do Starlight, abaixo do nome da área de
trabalho no menu ao lado esquerdo da interface.
:::
